# 머스테치

JSP 와 같은 템플릿 엔진

### 장점

1. 문법이 다른 템플릿 엔진보다 심플

2. 로직 코드가 없어서 View의 역할과 서버의 역할이 명확하게 분리
3. Mustache.js 와 Mustache.java 2가지가 있어, 하나의 문법으로 클라이언트 / 서버 템플릿을 모두 사용가능하다.

### 의존성 등록

```
implementation("org.springframework.boot:spring-boot-starter-mustache")
```

### 기본 템플릿 파일 생성


각 파일은 실제 다음 처럼 사용된다.

```
<!-- src/main/resources/templates/index.mustache -->

{{>layout/header}}
    <h1>기본 페이지</h1>
{{>layout/footer}}
```

### 글 등록 페이지

#### 글 등록 페이지 이동 버튼

```html
<!-- src/main/resources/templates/index.mustache -->

{{>layout/header}}
<div class="container">
    <h1>기본 페이지</h1>
    <a href="/posts/save" role="button" class="btn btn-primary">글 등록</a>
</div>
{{>layout/footer}}
```

#### 글 등록 페이지 Controller

```java
@GetMapping("/posts/save")
public String postsSave(){
    return "posts-save";
}
```

#### 글 등록 페이지 View

```html
<!-- src/main/resources/templates/posts-save.mustache -->

        <textarea class="form-control" id="content" rows="3"></textarea>
    </div>
    <button class="btn btn-primary" id="btn-save">글 등록</button>

{{>layout/footer}}
```

#### 글 등록 Script

```js
// src/main/resources/static/js/app/index.js
var main = {
    init : function() {
        var _this = this;
        $('#btn-save').on('click', function () {
            _this.save();
        });
    },
    save : function () {
        var data = {
            title: $('#title').val(),
            author: $('#author').val(),
            content: $('#content').val()
        };
        $.ajax({
            type: 'POST',
            url: '/api/v1/posts',
            dataType: 'json',
            contentType:'application/json; charset=utf-8',
            data: JSON.stringify(data)
        }).done(function() {
            alert('글이 등록되었습니다.');
            window.location.href = '/';
        }).fail(function (error) {
            alert(JSON.stringify(error));
        });
    },
};

main.init();
```
